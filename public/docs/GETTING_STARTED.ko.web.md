# BugIt 시작하기

BugIt는 대략적인 테스트 메모를 VS Code 안에서 검토된 버그 리포트로 바꿔 줍니다. VS Code와 GitHub Copilot이 설치된 Windows 11이 릴리스 검증을 마친 클라이언트 환경입니다.

## 시작하기 전에

- 최신 VS Code를 설치하고 GitHub Copilot에 로그인합니다.
- 릴리스 검증을 마친 Python 3.10부터 3.13까지의 인터프리터를 설치합니다.
- 계정 대시보드에서 BugIt를 다운로드하고 로컬 폴더에 압축을 풉니다.
- 라이선스 키, 토큰, 고객 데이터, 비공개 소스 코드는 채팅과 구성 파일에 포함하지 마세요.

## 활성화 및 구성

- 압축을 푼 BugIt 폴더를 신뢰할 수 있는 VS Code 작업 공간으로 엽니다.
- Copilot Chat에서 BugIt QA Agent를 선택하고 `Activate`를 입력합니다.
- 라이선스 키는 마스킹 처리된 터미널 프롬프트에만 입력합니다.
- `Begin setup`을 입력하고 팀이 사용하는 통합만 선택합니다.
- 티켓을 생성하기 전에 BugIt가 선택한 서비스와 프로젝트를 검증하도록 합니다.

## 연결 상태

- Jira Cloud와 Confluence Cloud는 가이드형 Atlassian Rovo MCP 경로를 사용하며, 브라우저 인증과 실시간 기능 확인이 필요합니다.
- Azure DevOps는 Microsoft의 조직 범위 원격 MCP 공개 미리 보기를 사용하며, 실시간 검증이 필요합니다.
- Sentry, GitHub, Linear, Notion은 각 서비스의 사전 조건과 실시간 확인을 통과하기 전까지 실험적 단계에 있습니다.
- 그 밖에 명시된 서비스에는 조직에서 제공하는 호환 MCP 서버가 필요합니다. BugIt는 설정 안내를 제공하지만 해당 서버를 배포하거나 테스트하지는 않습니다.

## 첫 리포트 작성

- 문제가 어디서, 얼마나 자주 발생하는지를 포함해 평이한 언어로 설명합니다.
- 재현 단계를 완성하는 데 필요한 질문에 답합니다.
- 미리 보기를 검토하되, 특히 비공개 데이터, 심각도, 프로젝트, 첨부 파일을 확인합니다.
- 대상과 최종 티켓이 올바른 경우에만 확정합니다.

## 도움 받기

먼저 BugIt 에이전트에서 `Check status` 또는 `Check readiness`를 실행하세요. 문제가 계속되면 BugIt 계정 대시보드에서 지원 티켓을 열되, 비밀 정보나 기밀 프로젝트 자료를 포함하지 마세요.
