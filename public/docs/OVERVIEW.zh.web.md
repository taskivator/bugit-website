# BugIt QA Agent — 概览

BugIt 是一款商用的 VS Code Copilot 代理，可将零散的测试记录转化为规范一致的缺陷报告。它先在你的工作区中本地起草，只有在你预览并确认之后，才会写入所连接的服务。

## 核心工作流

- 采集粗略的复现记录、日志、截图以及预期行为。
- 起草一份结构化报告，包含标题、严重程度、环境、复现步骤和证据。
- 在所连接的跟踪系统中搜索可能存在的重复项。
- 在执行任何外部写入之前，预览并确认目标位置与最终内容。
- 在缺陷修复经过复测后，补充验证性备注。

## 隐私与掌控

- BugIt 不会向 Taskivator 发送任何产品分析数据或工单遥测数据。
- 你所连接的 AI 提供方及已启用的集成，仅处理你选择发送给它们的内容。
- 许可与更新请求仅使用许可数据和一个单向的设备标识符，而不涉及工单内容。
- 试运行模式可防止随附的 Python 辅助工具执行写入，但你仍须审查外部的 MCP 操作。
- 配置文件中绝不能包含任何凭据值。

## 集成层级

- 引导式：通过 Atlassian Rovo MCP 接入 Jira Cloud 与 Confluence Cloud。
- 引导式公开预览：通过 Microsoft 的远程 MCP 服务接入 Azure DevOps。
- 实验性并支持实时验证：Sentry、GitHub、Linear 与 Notion。
- 仅提供设置指导：由组织自行提供的兼容服务器，用于对接其他跟踪系统、崩溃分析工具、测试管理、沟通及知识库服务。
- 自动化设置不支持：S3、Google Drive 与 Azure Blob 存储连接器。

## 发布范围

- BugIt 是当前已发布并持续维护的商用版本。
- Windows 11、VS Code、GitHub Copilot 以及 Python 3.10 至 3.13 是通过发布验证的运行环境。
- 完整的用户指南与概览提供英文及所有受支持语言的 PDF 版本，可在下方预览或下载。
