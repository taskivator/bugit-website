# Начало работы с BugIt

BugIt превращает черновые заметки о тестировании в проверенные отчёты об ошибках прямо в VS Code. Windows 11 с VS Code и GitHub Copilot — это клиентская конфигурация, для которой сертифицирован выпуск.

## Перед началом

- Установите последнюю версию VS Code и войдите в GitHub Copilot.
- Установите сертифицированный для выпуска интерпретатор Python 3.10–3.13.
- Загрузите BugIt из личного кабинета своей учётной записи и распакуйте его в локальную папку.
- Не помещайте лицензионные ключи, токены, данные клиентов и закрытый исходный код в чат и файлы конфигурации.

## Активация и настройка

- Откройте распакованную папку BugIt как доверенное рабочее пространство VS Code.
- В Copilot Chat выберите BugIt QA Agent и введите `Activate`.
- Вводите лицензионный ключ только в скрытом (замаскированном) запросе терминала.
- Введите `Begin setup` и выберите только те интеграции, которые использует ваша команда.
- Позвольте BugIt проверить выбранный сервис и проект перед созданием тикета.

## Состояние подключения

- Jira Cloud и Confluence Cloud используют путь с сопровождением через Atlassian Rovo MCP и требуют аутентификации в браузере, а также проверок возможностей в реальном времени.
- Azure DevOps использует публичный предпросмотр удалённого MCP от Microsoft с областью действия в рамках организации и требует проверки в реальном времени.
- Sentry, GitHub, Linear и Notion остаются экспериментальными, пока не будут выполнены предварительные требования их сервисов и проверки в реальном времени.
- Другим перечисленным сервисам требуется предоставленный организацией совместимый MCP-сервер. BugIt даёт рекомендации по настройке, но не поставляет и не тестирует такие серверы.

## Ваш первый отчёт

- Опишите проблему простым языком, указав, где она произошла и как часто.
- Ответьте на любые вопросы, необходимые для того, чтобы шаги воспроизведения были полными.
- Проверьте предпросмотр, особенно закрытые данные, серьёзность, проект и вложения.
- Подтверждайте только тогда, когда место назначения и итоговый тикет верны.

## Получить помощь

Сначала выполните `Check status` или `Check readiness` в агенте BugIt. Если проблема сохраняется, создайте запрос в поддержку из личного кабинета своей учётной записи BugIt, не включая в него секреты и конфиденциальные материалы проекта.
