# BugItを使い始める

BugItは、大まかなテストメモをVS Code内でレビュー済みのバグレポートへと変換します。VS CodeとGitHub Copilotを備えたWindows 11が、リリース認定済みのクライアント環境です。

## 始める前に

- 最新のVS Codeをインストールし、GitHub Copilotにサインインします。
- リリース認定済みのPython 3.10から3.13のインタープリターをインストールします。
- アカウントダッシュボードからBugItをダウンロードし、ローカルフォルダーに解凍します。
- ライセンスキー、トークン、顧客データ、非公開のソースコードは、チャットや設定ファイルに含めないようにします。

## 有効化と設定

- 解凍したBugItフォルダーを、信頼済みのVS Codeワークスペースとして開きます。
- Copilot ChatでBugIt QAエージェントを選択し、`Activate`と入力します。
- ライセンスキーは、マスクされたターミナルのプロンプトにのみ入力します。
- `Begin setup`と入力し、チームが利用する連携のみを選択します。
- チケットを起票する前に、選択したサービスとプロジェクトをBugItに検証させます。

## 接続状態

- Jira CloudおよびConfluence Cloudは、ガイド付きのAtlassian Rovo MCP経路を使用し、ブラウザ認証とライブ機能チェックを必要とします。
- Azure DevOpsは、Microsoftの組織スコープのリモートMCPパブリックプレビューを使用し、ライブ検証を必要とします。
- Sentry、GitHub、Linear、Notionは、各サービスの前提条件とライブチェックを満たすまでは実験的機能です。
- その他の記載サービスには、組織が提供する互換MCPサーバーが必要です。BugItはセットアップの手引きを提供しますが、それらのサーバーの提供やテストは行いません。

## 最初のレポート

- 問題を平易な言葉で説明し、発生した場所と頻度を含めます。
- 再現手順を完全なものにするために必要な質問に回答します。
- プレビューを確認します。特に非公開データ、重大度、プロジェクト、添付ファイルに注意してください。
- 書き込み先と最終的なチケットが正しい場合にのみ確定します。

## サポートを受ける

まずBugItエージェント内で`Check status`または`Check readiness`を実行してください。それでも問題が解決しない場合は、BugItアカウントダッシュボードからサポートチケットを起票してください。その際、機密情報や秘匿すべきプロジェクト資料は含めないでください。
